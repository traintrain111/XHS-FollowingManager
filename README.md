一个用于整理小红书已关注博主的 Chrome 扩展 🧺

当前版本的核心思路是：
- 在小红书搜索结果页勾选 `已关注`
- 用插件批量搜集博主
- 用 `二次搜集` 补抓主页简介
- 让插件自动分类，或手动微调标签

这份 README 面向试用同学，尽量按真实使用路径来写。

## 适合谁用 👀

- 想整理自己关注列表的人
- 想按主题给关注博主分类的人
- 想从一堆关注博主里快速找到某一类账号的人

## 当前支持的能力 ✅

- 扫描当前搜索结果页中的已关注博主
- 自动滚动搜集更多已加载结果
- 去重入库，同一博主多次扫描不会重复新增
- 保存博主昵称、主页链接、头像、备注、分类资料
- 支持收藏博主
- 支持搜索博主
- 支持手动创建/删除标签
- 支持在标签管理里查看某个标签下的博主
- 支持 `二次搜集`
说明：会慢速打开未分类博主主页，补抓简介等资料
- 支持二次搜集后确认是否自动分类
- 支持对单个博主手动补标签
- 支持对博主添加备注
- 支持在卡片上直接尝试 `已关注 / 关注` 切换
- 支持页面右上角常驻悬浮面板，可隐藏后再展开

## 当前不做的事 🙅

- 不接入 GPT/外部模型自动分类
- 不保证 100% 准确分类
- 不保证一次就抓到所有博主主页简介

## 项目技术栈 🛠️

- Vite
- CRXJS
- React
- TypeScript
- Chrome Extension Manifest V3

## 安装方式 🚀

### 1. 安装依赖 📦

```bash
npm install
```

### 2. 构建扩展 🔧

```bash
npm run build
```

构建完成后会生成 `dist/` 目录。

### 3. 在 Chrome 中加载扩展 🧩

1. 打开 `chrome://extensions/`
2. 打开右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择项目里的 `dist/` 目录

如果你更新了代码，需要：

1. 在扩展页点击一次 `重新加载`
2. 回到小红书页面再刷新一次

否则页面里的常驻面板和内容脚本可能还是旧版本。

## 标准使用路径 🧭

这是目前最推荐的使用方法。

### 路径 A：快速试用 ⚡

1. 打开小红书搜索结果页
2. 勾选 `已关注`
3. 点击 `扫描当前页`
4. 看看 `已搜集博主` 列表里是否已经入库

适合：

- 快速验证插件是否正常工作
- 少量页面内容的试扫

### 路径 B：正式整理 📚

1. 打开小红书搜索结果页
2. 勾选 `已关注`
3. 点击 `自动滚动搜集`
4. 等插件自动向下滚动并完成搜集
5. 检查 `已搜集博主`
6. 点击 `二次搜集`
7. 等它慢速补抓未分类博主主页资料
8. 二次搜集结束后，按弹窗提示决定是否自动分类
9. 进入 `管理标签` 查看分类结果

适合：

- 想一次整理更多博主
- 希望分类更准一些

## 页面上的主要按钮说明 🎛️

### `扫描当前页`

只扫描当前已经加载出来的结果。

特点：

- 速度快
- 风险低
- 适合试用

### `自动滚动搜集`

插件会自动滚动搜索结果页，尽量把更多结果加载出来并入库。

特点：

- 比手动下滑省事
- 比 `扫描当前页` 覆盖更多
- 如果搜索结果很多，优先用这个

### `管理标签`

切换到标签视图。

可以做的事：

- 新建标签
- 删除标签
- 点击标签查看该标签下的博主

### `收藏夹`

只看已收藏博主。

注意：

- 收藏只是一个筛选条件
- 进入收藏夹后，仍然按标签分组展示

### `二次搜集`

这是当前版本里提高分类命中率的关键步骤。

它会：

1. 找出当前未分类的博主
2. 慢速逐个打开主页
3. 尝试补抓简介等资料
4. 完成后询问你是否立即自动分类

注意：

- 它比普通扫描慢
- 它有一定风控风险
- 如果你担心账号稳定性，不建议连续高频使用

### `清空入库博主`

清空本地已入库的博主数据。

不会删除：

- 你手动创建的标签

会清掉：

- 博主列表
- 本地备注
- 收藏状态
- 自动抓到的资料

## 博主卡片说明 🪪

每张博主卡片当前会显示：

- 头像
- 昵称
- 主页链接
- `已关注 / 关注` 按钮
- 收藏按钮 `☆ / ★`
- 当前标签
- 备注输入框

### `已关注 / 关注`

点击后，插件会尝试打开该博主主页并执行站内关注或取消关注操作。

注意：

- 这是实操作，不是只改本地状态
- 如果小红书页面结构变化，可能失败

### `☆ / ★`

- `☆` 表示未收藏
- `★` 表示已收藏

收藏只存在本地，不会同步到小红书账号。

### 备注

备注会保存在本地，用于：

- 自己补充判断依据
- 后续搜索
- 辅助人工整理

## 标签管理怎么用 🏷️

点击 `管理标签` 后，可以看到标签列表。

### 新建标签

直接输入标签名后点击 `新建`。

目前限制：

- 最多 20 个标签
- 单个标签名最多 20 个字符

### 删除标签

删除后：

- 标签会被删除
- 该标签会同时从所有博主卡片上移除

### 查看标签下的博主

点击任一标签卡片后，下方会显示该标签下的博主卡片。

在这个区域里也可以：

- 看头像
- 看主页链接
- 看当前标签
- 加备注
- 收藏/取消收藏
- 执行关注切换

## 搜索功能 🔍

首页 `已搜集博主` 区域上方有搜索框。

可以搜索：

- 昵称
- 主页链接
- 标签
- 备注

适合拿来快速定位某位博主。

## 推荐的整理方式 💡

如果你准备给很多人试用，建议直接让他们照这个流程走：

1. 登录小红书网页版
2. 搜索一个主题词
3. 勾选 `已关注`
4. 点 `自动滚动搜集`
5. 点 `二次搜集`
6. 二次搜集结束后选择是否自动分类
7. 到 `管理标签` 里检查分类是否合理
8. 对少数不准的博主再手动改标签或加备注

如果一个主题词下搜集不全，可以换不同关键词再重复几轮，比如：

- 手帐
- 阅读
- 穿搭
- 搞笑
- 摄影

因为不同关键词搜到的已关注博主会不同，多轮下来更容易补全。

## 已知限制 ⚠️

### 1. 不能保证抓全所有关注博主

原因：

- 小红书是懒加载/无限流
- 页面结构可能变化
- 搜索词不同，能看到的已关注博主也不同

### 2. 自动分类不是 100% 准

当前分类主要依赖：

- 搜索结果卡片信息
- 二次搜集抓到的主页简介
- 备注

所以：

- 简介抽象时，容易分错
- 简介没抓到时，可能分不出来

### 3. 二次搜集有风控风险

因为它需要打开博主主页。

可能出现：

- 被要求安全验证
- 某轮没有抓到任何有效结果
- 账号短时异常

建议：

- 不要连续高频重复点击
- 先少量测试，再正式使用

### 4. 页面改版后，插件可能失效

如果小红书改了页面结构，以下能力都可能受影响：

- 搜索结果扫描
- 自动滚动搜集
- 简介抓取
- 关注按钮识别

## 常见问题 🙋

### 右上角没有出现插件面板怎么办？

先做这两步：

1. 重新加载扩展
2. 刷新当前小红书页面

### 为什么扫描不到博主？

先检查：

- 当前是不是小红书网页
- 当前是不是搜索结果页
- 是否勾选了 `已关注`
- 页面里是否真的已经加载出了用户结果

### 为什么自动滚动搜集后数量还是不多？

可能原因：

- 当前搜索词覆盖面太窄
- 页面还没加载出更多结果
- 当前页不是 `用户` 结果视图

建议换几个搜索词分别搜集。

### 为什么二次搜集后还是没有分类？

可能原因：

- 简介没有成功抓到
- 命中了安全验证页
- 简介内容过于抽象，无法稳定判断分类

这时建议：

- 看运行日志
- 手动补充备注
- 再人工打标签

### 为什么有些分类不准？

这是当前版本的正常现象。

建议把自动分类当作：

- 初步整理
- 节省一部分人工时间

而不是完全替代人工判断。

## 运行日志会显示什么 📝

运行日志主要用于帮助判断插件现在在做什么。

常见内容包括：

- 扫描新增了多少博主
- 自动滚动搜集是否完成
- 二次搜集开始/结束
- 某位博主是否抓到了简介摘要
- 是否命中安全验证页
- 自动分类后命中了什么标签

如果试用同学反馈问题，建议让他们连同运行日志一起发回来。

## 本地数据说明 💾

插件当前把数据保存在浏览器本地存储里。

包括：

- 已入库博主
- 标签
- 收藏状态
- 备注
- 已抓到的简介摘要

这意味着：

- 不会自动同步到别的电脑
- 卸载扩展或清本地数据后，可能丢失
