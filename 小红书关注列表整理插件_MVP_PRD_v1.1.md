# PRD - 小红书关注列表整理插件 MVP

**版本：** v1.1 (基于搜索筛选方案更新)  
**日期：** 2026-04-20  

---

## 一、产品目标

用户在小红书网页端，通过 Chrome 插件对关注的博主进行标签分类，并一键导出为飞书文档。由于网页版限制，本插件采用“搜索页筛选已关注”的路径进行博主数据搜集。

---

## 二、用户故事

> 作为一个关注了大量博主的小红书用户，我希望能给博主打标签分类，并导出到飞书文档，方便后续做标记和整理。

**核心路径：**
```
搜索并筛选「已关注」 → 扫描/搜集出现的博主 → 给博主打标签 → 导出到飞书文档
```

---

## 三、功能范围

### ✅ MVP 包含

| # | 功能 | 说明 |
|---|------|------|
| F1 | **搜集已关注博主** | 在搜索结果页（已勾选“已关注”筛选）抓取博主信息并去重存入本地库 |
| F2 | 标签管理 | 创建/删除标签（上限 20 个） |
| F3 | 给博主打标签 | 在插件列表内给已搜集的博主选择标签，支持多标签 |
| F4 | 飞书 OAuth 授权 | 获取写入飞书文档的权限 |
| F5 | 导出飞书文档 | 按标签分组导出，一级标题 + 带主页超链接的博主列表 |

---

## 四、功能详情

### F1 搜集已关注博主（核心变更）

- **操作路径：**
    1. 用户在小红书网页版搜索栏输入通用关键词（如“小红书”、“美食”、“穿搭”）。
    2. 用户手动点击「筛选」-「搜索范围」-「已关注」。
    3. 点击插件 Popup 中的**「扫描当前页博主」**按钮。
- **采集逻辑：**
    - Content Script 遍历当前搜索结果瀑布流卡片。
    - 提取卡片中的 `nickname`（作者昵称）和 `profile_url`（作者主页链接）。
    - **去重处理：** 将数据存入 `Chrome Storage Local`，若 `user_id` 已存在则更新（不覆盖已有标签），不存在则新增。
- **交互引导：** 提示用户“向下滚动页面以扫描更多博主”。

### F2 标签管理

- 支持新建标签（名称 1-20 字）、删除标签。
- 删除标签后，博主身上该标签自动移除。

### F3 给博主打标签

- 插件列表显示已搜集的博主，右侧提供「+ 标签」操作。
- 勾选即打标，再次点击取消。每个博主最多 5 个标签。
- 未打标签的博主自动归入「未分类」分组。

### F4 飞书 OAuth 授权

- 使用 `chrome.identity.launchWebAuthFlow` 完成飞书 OAuth 2.0 授权。
- 获取 `docs:doc:create` 和 `docs:doc:write` 权限。

### F5 导出飞书文档

- **文档标题：** 小红书关注列表整理 - [日期]
- **导出结构：**
    - `# 标签名` (Heading 1)
    - `• 博主昵称` (带有指向 profile_url 的超链接)

## 五、技术栈

| 模块 | 选型 |
|------|------|
| 插件规范 | Chrome Extension Manifest V3 |
| 前端框架 | React 18 + TypeScript |
| 样式处理 | Tailwind CSS |
| 构建工具 | Vite + CRXJS |
| 存储方案 | Chrome Storage Local |
| 飞书集成 | 飞书开放平台 OAuth 2.0 + Docs API |

## 六、验收标准

- [ ] 在搜索结果页开启“已关注”筛选后，能成功抓取瀑布流中的博主昵称与链接。
- [ ] 多次扫描不同搜索结果后，本地数据库能正确合并数据且不丢失已打的标签。
- [ ] 标签创建、删除以及给博主打标的交互逻辑正常。
- [ ] 导出到飞书的文档中，博主列表能按标签正确分组展示。
- [ ] 飞书文档中的超链接点击后能准确跳转至对应的博主个人主页。
