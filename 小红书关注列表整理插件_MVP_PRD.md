# PRD - 小红书关注列表整理插件 MVP

**版本：** v1.0 MVP  
**日期：** 2026-04-20  

---

## 一、产品目标

用户在小红书网页端，通过 Chrome 插件对关注的博主进行标签分类，并一键导出为飞书文档。文档以标签为一级标题，每个标题下为带主页超链接的博主列表。

---

## 二、用户故事

> 作为一个关注了大量博主的小红书用户，我希望能给博主打标签分类，并导出到飞书文档，方便后续做标记和整理。

**核心路径：**
```
抓取关注列表 → 给博主打标签 → 导出到飞书文档
```

---

## 三、功能范围

### ✅ MVP 包含

| # | 功能 | 说明 |
|---|------|------|
| F1 | 抓取关注列表 | 抓取博主昵称 + 主页链接，存本地 |
| F2 | 标签管理 | 创建/删除标签 |
| F3 | 给博主打标签 | 单个博主选择标签，支持多标签 |
| F4 | 飞书 OAuth 授权 | 获取写入飞书文档的权限 |
| F5 | 导出飞书文档 | 按标签分组导出，一级标题+超链接列表 |

### ❌ MVP 不包含

- 批量打标签
- 搜索 / 筛选
- 导出选项配置
- 标签颜色
- 取关建议、活跃度分析

---

## 四、功能详情

### F1 抓取关注列表

- 用户打开小红书网页版关注页（`www.xiaohongshu.com/user/profile/.../follows`）
- 点击插件 Popup 中的「抓取关注列表」按钮
- Content Script 自动滚动页面，抓取所有博主
- 每个博主保存：`nickname`、`profile_url`、`tags[]`
- 数据存入 Chrome Storage Local
- 重复抓取做合并，已有标签不覆盖

**异常：**
- 未登录小红书 → 提示"请先登录小红书网页版"
- 不在关注页 → 提示"请前往小红书关注页面"

---

### F2 标签管理

- 在 Popup 底部入口进入标签管理
- 支持新建标签（名称 1-20 字）、删除标签
- 删除标签后，博主身上该标签自动移除
- 标签上限：20 个

---

### F3 给博主打标签

- 关注列表每行博主右侧有「+ 标签」按钮
- 点击弹出已有标签列表，勾选即打标，再次点击取消
- 每个博主最多 5 个标签
- 未打标签的博主归入「未分类」

---

### F4 飞书 OAuth 授权

- 用户点击「导出到飞书」时触发
- 使用 `chrome.identity.launchWebAuthFlow` 完成飞书 OAuth 2.0
- 获取并存储 Access Token + Refresh Token
- 所需权限：`docs:doc:create`、`docs:doc:write`
- Token 过期自动刷新

---

### F5 导出飞书文档

**导出结构：**
```
（文档标题）小红书关注列表 - 2026/04/20

# 美食
• 小王美食        → 主页链接
• 吃货日记        → 主页链接

# 穿搭
• 时尚博主A       → 主页链接

# 未分类
• 某博主          → 主页链接
```

**实现方式（飞书 Docs API Block）：**
- 标签名 → `heading1`（block_type: 3）
- 博主 → `bullet`（block_type: 12）+ `text_element_style.link`

**完成后：**
- 弹出飞书文档直达链接，用户点击可直接打开

**异常处理：**
- 未授权 → 触发授权流程
- API 失败 → 提示错误原因，支持重试

## 六、技术栈

| 模块 | 选型 |
|------|------|
| 插件规范 | Chrome Extension Manifest V3 |
| 前端 | React 18 + TypeScript |
| 样式 | Tailwind CSS |
| 构建 | Vite + CRXJS |
| 存储 | Chrome Storage Local |
| 飞书集成 | 飞书开放平台 OAuth 2.0 + Docs API |

**目录结构：**
```
src/
├── background/     # Service Worker：OAuth 回调、飞书 API 调用
├── content/        # Content Script：抓取小红书关注页数据
├── popup/          # Popup UI（React）
└── shared/
    ├── storage.ts  # Chrome Storage 封装
    ├── feishu.ts   # 飞书 API 封装
    └── types.ts    # 数据类型
```

**核心数据结构：**
```typescript
interface Author {
  user_id: string;
  nickname: string;
  profile_url: string;
  tags: string[];        // 标签名数组
}

interface Tag {
  id: string;
  name: string;
}
```

## 七、飞书应用配置

需在飞书开放平台注册应用：
- 应用类型：网页应用
- OAuth 回调：`https://<extension-id>.chromiumapp.org/`
- 开通权限：`docs:doc:create`、`docs:doc:write`
